## üîÑ Reset Data

To start fresh:

1. Stop the application
2. Delete database files:
   ```bash
   rm credentials.db schooldata.db
   ```
3. Restart application (databases will be recreated)
4. Login with default superadmin credentials
5. Re-import data

## üêõ Common Test Issues

**Issue:** Students imported but can't login
- **Solution:** Make sure they login WITHOUT clicking ADMIN toggle

**Issue:** Grades not showing
- **Solution:** Check student_id in CSV matches actual user_id (S001, S002, etc.)

**Issue:** Schedule not loading
- **Solution:** Verify class name in CSV exactly matches class in system

**Issue:** Chat messages not real-time
- **Solution:** Ensure Socket.IO is working, refresh both browser windows

**Issue:** Homework not visible to students
- **Solution:** Check class name matches student's class exactly

## üí° Tips for Production Data

When preparing real school data:

1. **Export from existing system**
   - Most systems can export to CSV
   - Match column names to SchoolSync format

2. **Clean data before import**
   - Remove special characters
   - Standardize class names
   - Verify email formats
   - Check for duplicates

3. **Import in order**
   - Teachers first
   - Then students
   - Then subjects/classes
   - Then assign teachers to subjects
   - Finally schedules

4. **Test with small sample**
   - Import 5-10 records first
   - Verify everything works
   - Then import full dataset

5. **Keep backups**
   - Save original CSV files
   - Backup database before bulk import
   - Document import dates and sources
